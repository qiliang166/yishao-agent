# 04 — 步骤 1：内容获取

## 功能位置
项目详情页 → **「步骤 1 · 内容获取」**

## 前置条件
- 已创建项目
- 已进入项目详情页

## 操作说明

### 三种输入方式

#### 方式一：粘贴链接（推荐）
1. 将视频链接（如抖音、B站、YouTube 链接）粘贴到输入框
2. 选择 ASR 提供商和模型（在「厂商配置」中预先设置）
3. 点击 **「开始下载」** 按钮
4. 等待下载完成，进度条实时显示状态
5. 下载完成后自动进行字幕提取 + ASR 语音识别，LLM 交叉验证生成合并文本
6. 文本区上方出现来源切换标签：**合并版** | **语音识别** | **字幕**
7. 可手动编辑文本

#### 方式二：直接输入文本
1. 在编辑区直接粘贴或输入笔记文字
2. 点击 **「💾 保存」** 保存内容（刷新不丢失）
3. 点击 **「⚙ 整理文档」** 进入 AI 整理

#### 方式三：上传文本文件
1. 选择 `.txt` / `.md` / `.docx` 文件
2. 文件内容自动加载到编辑区
3. 点击 **「💾 保存」** 保存内容
4. 也可直接在编辑区手动编辑

### 来源切换标签
视频下载完成后，文本区上方显示三个标签：

| 标签 | 说明 |
|------|------|
| 合并版 | LLM 交叉验证合并字幕和语音识别结果（默认），差异处标注来源 |
| 语音识别 | 纯 ASR 语音转写结果 |
| 字幕 | 视频内嵌字幕原文（可能为空） |

### 视频播放校验
点击 **「📺 播放校验」** 按钮打开校验窗口：
- 左侧播放视频，右侧对照文本内容
- 窗口可拖曳移动、右下角可缩放
- 点击 **「💾 保存」** 保存文本修改
- 点击 **「🗑 清空」** 清空文本框

### 保存与清空
所有文本输入区域均提供 **💾 保存** 和 **🗑 清空** 按钮：
- 保存后内容持久化，刷新页面不丢失
- 保存成功时按钮变绿显示「✓ 已保存」

### 项目保存路径
每个项目可自定义存储路径（**📁 项目保存路径**）：
- 输入路径后点击「保存」生效
- 留空则使用全局默认路径（设置页的 save_path）
- 点击「📂」在文件管理器中打开当前路径

### 进度状态

| 状态 | 说明 |
|------|------|
| 🔄 下载中 | 正在下载视频，进度条显示百分比 |
| 📝 语音识别中 | 正在 ASR 语音转写 |
| 🔀 字幕+语音交叉验证中 | LLM 合并字幕和语音识别结果 |
| ✅ 完成 | 下载完成，文本已提取 |
| ❌ 失败 | 下载或提取失败，检查链接是否有效 |

## AI 生成结果
步骤 1 生成的结果提供三个操作按钮：
- **📥 保存到项目**：将 AI 整理结果直接保存到项目存储目录（同名文件自动加序号，不覆盖）
- **✓ 保存**：保存到数据库
- **✕ 清空**：清空编辑区

## 常见错误

| 问题 | 原因 | 解决 |
|------|------|------|
| 下载失败 | 链接无效或平台不支持 | 检查链接格式，确认平台在支持列表内 |
| 下载缓慢 | 网络问题 | 等待或尝试更换链接 |
| 字幕为空 | 视频无内嵌字幕 | 查看「语音识别」标签获取 ASR 结果 |
| ASR 转写失败 | ASR 提供商未配置或余额不足 | 检查「厂商配置」中的 ASR 提供商设置 |
| yt-dlp 报错 | 缺少 VC++ 运行时 | 安装 Microsoft Visual C++ Redistributable |
