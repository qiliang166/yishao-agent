## 视频播放校验修复

### 问题
视频校验弹窗中视频无法播放，`/api/video/file` 端点返回 403。

### 根因
Windows 路径大小写不敏感但 Python `str.startswith` 大小写敏感：
- `__file__` 返回 `D:\YISHAOAGENT\...` (大写 D)
- URL 参数解码后得到 `d:\YISHAOAGENT\...` (小写 d)
- `os.path.normpath()` 不改变盘符大小写
- 导致 `startswith` 检查失败

### 修复
- 使用 `os.path.normcase()` 统一转为小写后再比较
- 扩展允许路径范围：增加 `_get_global_save_path()` 根目录
- 后端: `backend/app.py` — `api_video_file` 端点

## PPT 规则系统：去掉预设，改为约束条文

### 问题
`design_rules` 中包含预设的 5 套配色方案、具体字重值（bold/600/400）等硬编码数据。用户指出这不合理 — 视觉样式应该从上传的 .pptx 模板中实际提取，栏目配置的 rules 应定义**约束规则**而非**预设值**。

### 核心改动
```
之前：rules 预设 5 套配色 → AI 从预设中"选最接近的"
现在：rules 是约束条文 → AI 从 .pptx 提取实际样式 → 按规则审视优化 → 生成完整 prompt+skill
```

### 具体变更

**database.py:**
- `RULES_COL4`/`RULES_COL5` 的 `design_rules` 重写：
  - 去掉：`color_schemes`(5套预设)、`font_weights`(bold/600/400)、`slide_ratio`、`branding`
  - 改为 5 条约束条文：`color_discipline`、`font_discipline`、`layout_discipline`、`slide_discipline`、`branding_discipline`
- checklist P0-2/P1-4 更新为引用约束条文而非预设值
- clarification_questions q5 从"想要哪套主题色"改为"配色风格是否符合内容主题"
- `analysis_rules` 重写：要求 AI 从模板提取实际颜色/字体，不预设配色方案
- 新增迁移 UPDATE：已有 DB 的 col4/col5 rules 自动升级（通过 `LIKE '%color_schemes%'` 检测旧版本）
- 种子数据 prompt 更新为使用约束语言

**app.py:**
- `extract_pptx_structure()` 增强：
  - 提取占位符字体信息（name/size_pt/bold）→ `fonts_used` 汇总
  - 提取文字颜色（RGB）→ `colors_used` 汇总
  - 新增 `master_layouts`（母版版式列表）
- `analyze_template` system_prompt 更新：
  - 要求 AI 从提取的 PPTX 结构获取实际配色/字体
  - 按 design_rules 约束条文审视模板样式
  - 输出格式去掉 `color_scheme` 字段
- `_build_prompt_from_rules()` 更新：
  - 引用 `color_discipline`/`font_discipline` 等约束条文
  - 不再引用 `color_schemes`/`font_weights` 预设值

## PPT 文字错位修复 + 字体修复 + 站点名称动态化 + 模板智能解析增强

### 用户反馈
1. 导出的 PPT 文字错位
2. 下载的 PPT 字体是 PingFang SC（苹方），不是微软雅黑
3. 健康检查的 app 名称「一勺笔录(SOP)智能体」是硬编码，需要可配置
4. 模板智能解析需要从 .pptx 母版提取实际主题颜色和字体

### 根因分析

**文字错位**: `_normalize_formatting()` 强制将所有段落设为 `PP_ALIGN.LEFT`，覆盖了幻灯片版式原有的对齐设置（标题居中、内容左对齐等）。修复：移除对齐强制修改，只设置字体。

**PingFang SC 字体**: python-pptx 的 `run.font.name = "Microsoft YaHei"` 只设置 Latin（西文）字体，不设置 EA（东亚/中文）字体。中文字符仍使用模板主题的默认东亚字体（PingFang SC）。修复：通过 lxml 操作 run XML，同时设置 `<latin>` 和 `<ea>` 元素的 typeface。

**站点名称硬编码**: 3 处硬编码「一勺笔录(SOP)智能体」。修复：新增 `DEFAULT_SITE_NAME` 常量 + `_get_site_name()` 函数从 settings 表读取，可通过 `PUT /api/settings` 配置 `site_name`。

**主题颜色/字体提取为空**: 默认模板使用主题级样式（非逐段落显式设置），`extract_pptx_structure()` 仅在占位符的 run 中查找，未读取母版主题 XML。修复：遍历 slide masters 的 XML，提取 `srgbClr`/`sysClr` 颜色和 `latin`/`ea` 字体。

### 文件修改
| # | 文件 | 操作 |
|---|------|------|
| 1 | `backend/services/ppt_service.py` | 新增 `_set_font_all()` 同时设置 Latin + EA 字体；`_normalize_formatting()` 移除 LEFT 对齐强制；添加 `from lxml import etree` + `_NS` 常量 |
| 2 | `backend/app.py` | 新增 `DEFAULT_SITE_NAME` + `_get_site_name()`；3 处硬编码替换；`extract_pptx_structure()` 增强：遍历母版 XML 提取 `theme_colors` 和 `theme_fonts` |

## 2026-06-24 模板自适应排版间距系统

### 目标
将 PPT 间距系统从硬编码绝对 EMU 值改为从模板实际排版数据按比例计算，实现真正的"模板自适应"。

### 架构：四层数据流
```
栏目JSON规则 (typography_spec: 提取清单 + 规范兜底值)
    ↓
代码提取 (_extract_typography: 从模板XML继承链提取实际字号/行高)
    ↓ (有缺漏的)
AI分析 (analyze_template: 按规范补全缺失值)
    ↓
模板 typography_profile (完整无缺漏，存入templates表)
    ↓
PPT生成 (_compute_spacing_from_profile: 字号×比例=实际EMU间距)
```

### 核心改动

**1. `backend/services/ppt_service.py`**
- 新增 `_extract_typography(prs)`: 遍历所有 run.font.size + master bodyStyle/titleStyle XML 继承链，提取 title_font_size_pt / body_font_size_pt / line_height_ratio
- 新增 `_resolve_run_size(run, ns)`: 沿 run.rPr → para.defRPr 链解析继承字号
- 新增 `_resolve_line_height(prs, ns)`: 从 master bodyPr.normAutofit 或 lvl1pPr.lnSpc 提取行高倍率
- 新增 `_compute_spacing_from_profile(profile)`: 将 typography_profile 转换为实际 EMU 间距值
- `_normalize_spacing` 间距比例: element_gap = 1/3 line_height, text_line_gap = 1/4, peer_tolerance = 3/20, title_gap = 1/5 title_size
- `generate_ppt()`: 优先从 templates.typography_profile 读取并计算间距，无 profile 时 fallback 旧默认值

**2. `backend/database.py`**
- templates 表新增 `typography_profile` 列
- col4/col5 design_rules 新增 `typography_spec`（3 个提取项 + fallback + rationale）
- 现有 DB 迁移：为已有 col4/col5 规则的 DB 自动添加 typography_spec

**3. `backend/app.py`**
- `extract_pptx_structure()` 返回增加 `typography_extracted` 字段
- `analyze_template` system_prompt 增加排版属性补全指令（typography_spec + typography_extracted → AI 补全 → typography_profile）
- `analyze_template` 响应解析增加 typography_profile，存入 templates 表

**间距比例设计依据**
| 比例 | 值 | 依据 |
|------|-----|------|
| element_gap / line_height | 1/3 | 国开标准层级缩进≥1字符；ISO 24896 视觉元素一致间距 |
| text_line_gap / line_height | 1/4 | SJ/T 11841.6.1 视觉舒适度 |
| title_gap / title_size | 1/5 | 标题字号大，比例缩小避免间距过大 |
| peer_tolerance / line_height | 3/20 | ISO 24896 同行元素对齐容差 |

| 2 | `backend/database.py` | templates 表新增 typography_profile 列；col4/col5 design_rules 新增 typography_spec；DB 迁移 |
