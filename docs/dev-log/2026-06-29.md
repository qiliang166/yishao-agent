## PPT 颜色硬编码全面清理（两层防御）

### 问题
VI 规格 markdown 文件和 prompt 文件中包含硬编码 hex 色值（`#1A365D`、`#E67E22`、`#e2e8f0` 等），LLM 在生成 HTML 时会直接复制这些 hex 值到输出中，绕过 `{{placeholder}}` 颜色变量系统，导致配色方案切换失效。

### 根因
`_load_scenario_file()` 和 prompt 加载链路将 markdown 文件原文送入 LLM prompt，LLM 从 prompt 示例中学习并复制 hex 色值。VI 文件中的颜色参考表、示例代码片段都成为了硬编码 hex 的来源。

### 修复方案：两层防御

**Layer 1 — Prompt 源码清理（根因修复）**
将所有送入 LLM 的 prompt/VI/scenario 文件中的硬编码 hex 替换为 `{{placeholder}}` 或 `${placeholder}` 变量语法：
- `#1A365D`→`{{primary}}`、`#E67E22`→`{{accent}}`、`#e2e8f0`→`rgba({{text_rgb}}, 0.12)`
- `#27ae60`→`{{semantic_positive}}`、`#c0392b`→`{{semantic_negative}}`
- `#333`→`{{text}}`（或 `${text}`）、`#6366f1`→`${primary}`
- 保留 `#ffffff`：封面/表格头/网络节点等暗色背景上的白色文字，符合铁律 #5
- 保留 `#ff0000`/`#00ff00`：iron-laws.md 中的故意反例

**Layer 2 — 代码后处理守卫**
`ppt_service.py` 新增 `_auto_fix_hardcoded_hex()` 函数（line 1141），在每页 HTML 生成后、颜色变量解析前执行：
- 从 active_scheme 构建反向 hex→placeholder 映射表
- 扫描 LLM 输出中的所有 hex 值
- 自动替换为 `{{placeholder}}`（排除 `#ffffff`）
- 记录替换数量 + 未知 hex 警告日志

调用点：`ppt_service.py:2416`（`_fix_llm_html_errors` 之后，`_resolve_color_vars` 之前）

## CSS 变量方案 — 色系一致性根本性修复

### 问题
颜色色系在 regenerate、splice、recolor 三个操作中不一致。根本原因是 hex 替换方案：每个幻灯片的 hex 值需要逐个检测和替换，容易遗漏或检测错误。

### 方案
HTML 输出改用 CSS 变量引用（`var(--primary)`）+ 全局 `:root` 块定义实际色值。换色只需替换 `:root` 块 — O(1) 操作，所有 `var(--)` 引用自动生效。

### 改动清单

**`backend/services/ppt_service.py`**
- `_resolve_color_vars` (line 1432): 新增 `css_vars` 参数。`css_vars=False`（默认）输出 hex 值用于 LLM 提示词构建；`css_vars=True` 输出 `var(--key)` 用于最终 HTML
- `_build_root_vars` (line 3466): 新增函数，从 scheme_data 生成 `:root { }` CSS 块，定义全部 13 个颜色变量及 RGB 变体
- `_assemble_html_deck` (line 3519): style 块注入 `:root` 变量定义
- HTML 输出调用点: 传入 `css_vars=True`

**`backend/app.py`**
- recolor 端点 (line 3042): 检测 `:root {` 判断新旧格式。新格式替换 `:root` 块；旧格式回退逐页 hex 替换
- regenerate 端点: 传入 `css_vars=True`；修复 deck 组装后缺少 `_resolve_color_vars` 的问题
- splice 端点: 传入 `css_vars=True`；自动注入 `:root` 块处理新旧格式过渡

### 向后兼容
- 旧 run（index.html 无 `:root` 块）：recolor 走旧 `_recolor_slide_html` 逻辑
- LLM 提示词构建：`_resolve_color_vars` 默认 `css_vars=False`，继续输出 hex 值
- `_auto_fix_hardcoded_hex` → `_resolve_color_vars` 管道不变

### 技术细节
- 13 个颜色变量/色系：primary, secondary, accent, background, text, card_bg + 5 chart_colors + 2 semantic (positive, negative)
- 5 套色系：deep-blue, chinese-red, jade-green, tech-purple, dark-gold
- 每个变量含 5 个 CSS 自定义属性：`--{key}`, `--{key}-r`, `--{key}-g`, `--{key}-b`, `--{key}-rgb`
- Splice 过渡处理：新 CSS-variable 幻灯片拼接到旧格式 HTML 时自动注入 `:root` 块

- curl 旧格式 recolor：hex_replace 方法正常
- curl 新格式 recolor：css_vars 方法正常，`:root` 块正确替换
- 664 `var(--)` 引用在换色后保持不变，仅 `:root` 块值更新
- 前端 build：无错误
